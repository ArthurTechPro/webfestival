# Servicio de Email - WebFestival

## Descripción

El servicio de email de WebFestival proporciona funcionalidad completa para el envío de notificaciones por correo electrónico a los usuarios de la plataforma. Implementa los requisitos 12.1-12.4 del sistema de notificaciones.

## Características Principales

### ✅ Proveedores Soportados
- **SendGrid**: Proveedor principal de email transaccional
- **Resend**: Proveedor alternativo moderno
- **Selección automática**: Basada en variable de entorno `EMAIL_SERVICE`

### ✅ Sistema de Reintentos
- **Máximo de reintentos**: 3 intentos por email
- **Delay incremental**: 1s, 2s, 3s entre reintentos
- **Manejo de errores**: Logging detallado y recuperación automática

### ✅ Cola de Procesamiento
- **Envío masivo**: Procesamiento en lotes de 10 emails
- **Rate limiting**: Pausa de 500ms entre lotes
- **Prevención de sobrecarga**: Control de concurrencia

## Configuración

### Variables de Entorno

```bash
# Proveedor de email (sendgrid | resend)
EMAIL_SERVICE="sendgrid"

# Configuración SendGrid
SENDGRID_API_KEY="your-sendgrid-api-key"
SENDGRID_FROM_EMAIL="noreply@webfestival.com"

# Configuración Resend
RESEND_API_KEY="your-resend-api-key"
RESEND_FROM_EMAIL="noreply@webfestival.com"
```

## Uso del Servicio

### Importación

```typescript
import { getEmailService } from '../services/email.service';

const emailService = getEmailService();
```

### Métodos Principales

#### 1. Recordatorio de Fecha Límite (Requisito 12.1)

```typescript
await emailService.sendDeadlineReminder(
  'user@example.com',
  'Juan Pérez',
  'Concurso de Fotografía Urbana 2024'
);
```

**Cuándo se usa**: 48 horas antes de la fecha límite de un concurso inscrito.

#### 2. Evaluación Completada (Requisito 12.2)

```typescript
await emailService.sendEvaluationComplete(
  'user@example.com',
  'María García',
  'Concurso de Video Digital 2024'
);
```

**Cuándo se usa**: Cuando un jurado completa la evaluación de los medios de un participante.

#### 3. Resultados Publicados (Requisito 12.3)

```typescript
await emailService.sendResultsPublished(
  'user@example.com',
  'Carlos López',
  'Concurso de Música Electrónica 2024'
);
```

**Cuándo se usa**: Cuando se publican los resultados oficiales de un concurso.

#### 4. Nuevo Concurso (Requisito 12.4)

```typescript
await emailService.sendNewContestNotification(
  'user@example.com',
  'Ana Martínez',
  'Nuevo Concurso de Arte Digital 2024'
);
```

**Cuándo se usa**: Cuando un administrador crea un nuevo concurso.

### Métodos de Envío Masivo

#### Notificar a Todos los Participantes

```typescript
const participants = [
  { email: 'user1@example.com', name: 'Usuario 1' },
  { email: 'user2@example.com', name: 'Usuario 2' }
];

await emailService.notifyAllParticipants(participants, 'Concurso de Fotografía');
```

#### Notificar a Todos los Usuarios

```typescript
const users = [
  { email: 'user1@example.com', name: 'Usuario 1' },
  { email: 'user2@example.com', name: 'Usuario 2' }
];

await emailService.notifyAllUsers(users, 'Nuevo Concurso Disponible');
```

### Envío Personalizado

```typescript
const emailData = {
  to: 'user@example.com',
  subject: 'Asunto personalizado',
  html: '<h1>Contenido HTML</h1>',
  text: 'Contenido en texto plano'
};

const success = await emailService.sendEmail(emailData);
```

## Mejores Prácticas

### 1. Manejo de Errores

```typescript
try {
  const success = await emailService.sendDeadlineReminder(email, name, contest);
  if (!success) {
    // Manejar fallo de envío
    console.error('No se pudo enviar el recordatorio');
  }
} catch (error) {
  // Manejar error de configuración
  console.error('Error de configuración del servicio:', error);
}
```

### 2. Validación de Datos

```typescript
// Validar email antes de enviar
if (!email || !email.includes('@')) {
  throw new Error('Email inválido');
}

// Validar nombre de usuario
if (!userName || userName.trim().length === 0) {
  throw new Error('Nombre de usuario requerido');
}
```

### 3. Rate Limiting

El servicio implementa automáticamente rate limiting para envío masivo:
- Lotes de 10 emails máximo
- Pausa de 500ms entre lotes
- Prevención de sobrecarga del proveedor

## Troubleshooting

### Problemas Comunes

1. **Error: "SENDGRID_API_KEY es requerido"**
   - Verificar que la variable de entorno esté configurada
   - Verificar que la API key sea válida

2. **Emails no se envían**
   - Verificar configuración del proveedor
   - Revisar logs para errores específicos
   - Verificar límites de rate limiting del proveedor

3. **Templates no se renderizan correctamente**
   - Verificar que los datos del template estén completos
   - Revisar la sintaxis HTML del template

### Debugging
