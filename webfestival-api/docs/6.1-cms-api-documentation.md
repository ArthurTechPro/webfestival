# Documentación API CMS - WebFestival

## Tarea 6.1: APIs del CMS Principal

### 📋 Resumen

Esta documentación describe la implementación completa del sistema CMS unificado para WebFestival, que proporciona endpoints CRUD para la gestión de contenido, configuración, SEO, métricas y taxonomía.

### 🎯 Objetivos Cumplidos

- ✅ Endpoints CRUD para tabla principal de contenido
- ✅ APIs específicas para configuración, SEO y métricas
- ✅ Endpoints para gestión de taxonomía (categorías y etiquetas)
- ✅ API para plantillas dinámicas por tipo de contenido
- ✅ Endpoints para preview y publicación de cambios
- ✅ Sistema de filtros y búsqueda optimizado

### 🏗️ Arquitectura

El sistema CMS está compuesto por:

1. **Controlador** (`cms.controller.ts`) - Maneja las peticiones HTTP
2. **Servicio** (`cms.service.ts`) - Lógica de negocio y operaciones de base de datos
3. **Esquemas** (`cms.schemas.ts`) - Validación de datos con Zod
4. **Rutas** (`cms.routes.ts`) - Definición de endpoints y middleware

### 📊 Modelo de Datos

#### Tipos de Contenido Soportados

- **`pagina_estatica`** - Contenido estático para la landing page
- **`blog_post`** - Artículos del blog de la comunidad
- **`seccion_cms`** - Secciones personalizables del CMS

#### Estados de Contenido

- **`BORRADOR`** - Contenido en desarrollo
- **`PUBLICADO`** - Contenido visible públicamente
- **`ARCHIVADO`** - Contenido archivado
- **`PROGRAMADO`** - Contenido programado para publicación futura

## 🌐 Endpoints del API

### Rutas Públicas (Solo Lectura)

#### 1. Obtener Contenido con Filtros
```http
GET /api/cms/content
```

**Parámetros de consulta:**
- `tipo` (string, opcional) - Tipo de contenido
- `categoria` (string, opcional) - Filtrar por categoría
- `etiqueta` (string, opcional) - Filtrar por etiqueta
- `autor` (string, opcional) - Filtrar por autor
- `estado` (string, opcional) - Estado del contenido
- `busqueda` (string, opcional) - Búsqueda en título, contenido y resumen
- `activo` (boolean, opcional) - Filtrar por contenido activo
- `destacado` (boolean, opcional) - Filtrar por contenido destacado
- `page` (number, opcional) - Página actual (default: 1)
- `limit` (number, opcional) - Elementos por página (default: 10, max: 100)
- `sort_by` (string, opcional) - Campo para ordenar (default: created_at)
- `sort_order` (string, opcional) - Orden asc/desc (default: desc)

**Respuesta exitosa:**
```json
{
  "success": true,
  "data": {
    "contenido": [...],
    "total": 50,
    "page": 1,
    "totalPages": 5,
    "hasNext": true,
    "hasPrev": false
  },
  "message": "Contenido obtenido exitosamente"
}
```

#### 2. Obtener Contenido por Slug
```http
GET /api/cms/content/:slug
```

**Parámetros:**
- `slug` (string) - Slug único del contenido

**Parámetros de consulta:**
- `include` (boolean, opcional) - Incluir relaciones (default: true)

#### 3. Obtener Categorías
```http
GET /api/cms/categories
```

**Parámetros de consulta:**
- `tipo` (string, opcional) - Filtrar por tipo de contenido

#### 4. Obtener Etiquetas con Autocompletado
```http
GET /api/cms/tags
```

**Parámetros de consulta:**
- `query` (string, requerido) - Texto de búsqueda
- `limit` (number, opcional) - Límite de resultados (default: 10, max: 20)

#### 5. Obtener Tipos de Contenido
```http
GET /api/cms/content-types
```

#### 6. Obtener Plantilla por Tipo
```http
GET /api/cms/content-template/:tipo
```

**Parámetros:**
- `tipo` (string) - Tipo de contenido (pagina_estatica, blog_post, seccion_cms)

### Rutas Protegidas (Requieren Autenticación)

#### 7. Obtener Métricas de Contenido

#### 8. Generar Preview de Contenido

Authorization: Bearer <token>

## 🔐 Seguridad y Autenticación

### Niveles de Acceso

1. **Público** - Endpoints de solo lectura sin autenticación
2. **Autenticado** - Requiere token JWT válido
3. **CONTENT_ADMIN** - Requiere rol CONTENT_ADMIN o ADMIN

### Validación de Datos

Todos los endpoints utilizan esquemas Zod para validación robusta:

- **Validación de tipos** - Tipos de datos correctos
- **Validación de longitud** - Límites de caracteres
- **Validación de formato** - URLs, fechas, slugs
- **Validación de reglas de negocio** - Lógica específica del dominio

### Manejo de Errores

Respuestas de error estandarizadas:

```json
{
  "success": false,
  "message": "Descripción del error",
  "errors": [...] // Detalles de validación (opcional)
}
```

Códigos de estado HTTP:
- `400` - Datos inválidos
- `401` - No autenticado
- `403` - Sin permisos
- `404` - Recurso no encontrado
- `409` - Conflicto (ej: slug duplicado)
- `500` - Error interno del servidor

El sistema genera automáticamente slugs únicos basados en el título:

- Conversión a minúsculas

Cada tipo de contenido tiene plantillas configurables:

```json
{

  "campos": [
    {
      "nombre": "titulo",
      "tipo": "text",
      "requerido": true
    },
    {
      "nombre": "contenido",
      "tipo": "wysiwyg",
      "requerido": true
    }
  ],
  "configuracion": {
    "permite_comentarios": true,
    "seo_automatico": true
  }
}
```

### 3. SEO Automático

- Generación automática de títulos SEO (60 caracteres)
- Creación de meta descripciones desde el resumen (160 caracteres)
- Soporte para meta tags personalizados
- Structured data para motores de búsqueda

### 4. Métricas en Tiempo Real

Seguimiento automático de:
- Vistas de página
- Likes y reacciones
- Conteo de comentarios
- Shares en redes sociales
- Fecha de primera publicación
- Última vista registrada

### 5. Sistema de Taxonomía Flexible

- **Categorías** - Clasificación jerárquica
- **Etiquetas** - Etiquetado libre con autocompletado
- Búsqueda y filtrado por taxonomía
- Gestión independiente por tipo de contenido

### 6. Preview Seguro

Sistema de preview para contenido no publicado:
- URLs temporales con tokens
- Acceso restringido a CONTENT_ADMIN
- Visualización sin afectar métricas públicas

## 🧪 Ejemplos de Uso

### Crear un Post de Blog

```javascript
const response = await fetch('/api/cms/content', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer ' + token,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    tipo: 'blog_post',
    titulo: 'Anunciamos el WebFestival 2024',
    contenido: '<p>Estamos emocionados de anunciar...</p>',
    resumen: 'El evento más esperado del año está aquí',
    estado: 'BORRADOR'
  })
});
```

### Buscar Contenido

```javascript
const response = await fetch('/api/cms/content?' + new URLSearchParams({
  busqueda: 'webfestival',
  tipo: 'blog_post',
  estado: 'PUBLICADO',
  page: '1',
  limit: '10'
}));
```

### Actualizar SEO

```javascript
const response = await fetch(`/api/cms/content/${contentId}/seo`, {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer ' + token,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    seo_titulo: 'WebFestival 2024 - El Concurso Multimedia Más Grande',
    seo_descripcion: 'Participa en WebFestival 2024 y muestra tu talento',
    seo_keywords: ['webfestival', 'concurso', 'multimedia', '2024']

```
