## Newsletter APIs

### Suscripción al Newsletter

#### POST `/api/newsletter/subscribe`
Permite a los usuarios suscribirse al newsletter.

**Acceso:** Público

**Body:**
```json
{
  "email": "usuario@example.com"
}
```

**Respuesta exitosa (201):**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "email": "usuario@example.com",
    "activo": true,
    "confirmado": false,
    "token_confirmacion": "abc123...",
    "fecha_suscripcion": "2025-10-05T02:00:00.000Z"
  },
  "message": "Suscripción creada exitosamente. Revisa tu email para confirmar la suscripción."
}
```

### Confirmar Suscripción

#### POST `/api/newsletter/confirm`
Confirma una suscripción al newsletter usando el token enviado por email.

**Acceso:** Público

**Body:**
```json
{
  "token": "abc123..."
}
```

### Cancelar Suscripción

#### POST `/api/newsletter/unsubscribe`
Cancela una suscripción al newsletter.

**Acceso:** Público

**Body:**
```json
{
  "email": "usuario@example.com",
  "token": "abc123..." // Opcional
}
```

### Gestión de Suscriptores (CONTENT_ADMIN)

#### GET `/api/newsletter/subscribers`
Obtiene lista de suscriptores con filtros.

**Acceso:** CONTENT_ADMIN

**Query Parameters:**
- `activo` (boolean): Filtrar por estado activo
- `confirmado` (boolean): Filtrar por confirmación
- `fecha_desde` (datetime): Fecha de inicio
- `fecha_hasta` (datetime): Fecha de fin
- `page` (number): Página (default: 1)
- `limit` (number): Límite por página (default: 10)

#### PUT `/api/newsletter/subscribers/:id`
Actualiza un suscriptor.

**Acceso:** CONTENT_ADMIN

**Body:**
```json
{
  "activo": true,
  "confirmado": true
}
```

#### GET `/api/newsletter/stats`
Obtiene estadísticas del newsletter.

**Acceso:** CONTENT_ADMIN

**Respuesta:**
```json
{
  "success": true,
  "data": {
    "total_suscriptores": 150,
    "suscriptores_activos": 140,
    "suscriptores_confirmados": 120,
    "nuevos_este_mes": 25,
    "tasa_confirmacion": 80.0
  }
}
```

## Contenido Educativo APIs

### Crear Contenido Educativo

#### POST `/api/educational-content`
Crea nuevo contenido educativo.

**Acceso:** CONTENT_ADMIN

**Body:**
```json
{
  "tipo": "tutorial",
  "categoria_multimedia": "fotografia",
  "titulo": "Tutorial de Fotografía de Paisajes",
  "contenido": "Contenido completo del tutorial...",
  "resumen": "Aprende técnicas de fotografía de paisajes",
  "nivel": "intermedio",
  "tiempo_lectura": 15,
  "tags": ["paisajes", "naturaleza", "composicion"],
  "imagen_destacada": "https://example.com/imagen.jpg",
  "recursos_adicionales": [
    {
      "titulo": "Video complementario",
      "url": "https://example.com/video",
      "tipo": "video"
    }
  ],
  "estado": "publicado"
}
```

**Tipos válidos:**
- `tutorial`: Tutorial paso a paso
- `articulo`: Artículo informativo
- `guia`: Guía completa
- `inspiracion`: Contenido inspiracional

**Categorías multimedia:**
- `fotografia`: Fotografía
- `video`: Video
- `audio`: Audio
- `cine`: Cortos de cine
- `general`: General

**Niveles:**
- `principiante`: Nivel básico
- `intermedio`: Nivel intermedio
- `avanzado`: Nivel avanzado

### Obtener Contenido Educativo

#### GET `/api/educational-content`
Obtiene contenido educativo con filtros.

**Acceso:** Público

**Query Parameters:**
- `tipo`: Tipo de contenido
- `categoria_multimedia`: Categoría multimedia
- `nivel`: Nivel de dificultad
- `estado`: Estado del contenido
- `autor_id`: ID del autor
- `tags`: Array de etiquetas
- `busqueda`: Búsqueda de texto libre
- `fecha_desde`: Fecha de inicio
- `fecha_hasta`: Fecha de fin
- `destacado`: Contenido destacado
- `page`: Página
- `limit`: Límite por página
- `orden`: Orden (`reciente`, `popular`, `titulo`, `tiempo_lectura`)

#### GET `/api/educational-content/:id`
Obtiene contenido educativo por ID.

**Acceso:** Público

### Recomendaciones Personalizadas

#### GET `/api/educational-content/recommendations`
Obtiene recomendaciones personalizadas basadas en el historial del usuario.

**Acceso:** Usuario autenticado

**Query Parameters:**
- `categoria_multimedia`: Filtrar por categoría
- `nivel`: Filtrar por nivel
- `limit`: Número de recomendaciones (default: 10)
- `excluir_leidos`: Excluir contenido ya leído (default: true)

**Respuesta:**
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "titulo": "Tutorial Avanzado de Composición",
      "tipo_educativo": "tutorial",
      "categoria_multimedia": "fotografia",
      "nivel": "avanzado",
      "tiempo_lectura": 20,
      "tags": ["composicion", "avanzado"],
      "recommendation_score": 8.5,
      "metricas": {
        "vistas": 150,
        "likes": 25
      }
    }
  ]
}
```

### Tracking de Métricas

#### POST `/api/educational-content/track-view`
Registra una vista de contenido educativo.

**Acceso:** Público (opcional autenticado)

**Body:**
```json
{
  "contenido_id": 1,
  "tiempo_lectura": 300,
  "porcentaje_leido": 85
}
```

#### GET `/api/educational-content/metrics`
Obtiene métricas de contenido educativo.

**Acceso:** CONTENT_ADMIN

**Query Parameters:**
- `tipo`: Tipo de contenido
- `categoria_multimedia`: Categoría multimedia
- `periodo`: Período (`semana`, `mes`, `trimestre`, `año`)
- `fecha_desde`: Fecha de inicio
- `fecha_hasta`: Fecha de fin
- `limit`: Límite de resultados

**Respuesta:**
```json
{
  "success": true,
  "data": {
    "resumen": {
      "total_contenido": 50,
      "total_vistas": 1250,
      "total_likes": 180,
      "promedio_tiempo_lectura": 12
    },
    "contenido_mas_visto": [...],
    "contenido_mas_liked": [...],
    "estadisticas_por_categoria": [
      {
        "categoria": "fotografia",
        "total_contenido": 20,
        "total_vistas": 500,
        "total_likes": 75
      }
    ]
  }
}
```

## Digest APIs

### Generar Digest Semanal

#### POST `/api/newsletter/generate-digest`
Genera un digest semanal para el newsletter.

**Acceso:** CONTENT_ADMIN

**Body:**
```json
{
  "fecha_inicio": "2025-09-28T00:00:00.000Z",
  "fecha_fin": "2025-10-05T23:59:59.999Z",
  "incluir_concursos": true,
  "incluir_contenido_educativo": true,
  "incluir_ganadores": true,
  "max_contenido": 5,
  "max_concursos": 3
}
```

**Respuesta:**
```json
{
  "success": true,
  "data": {
    "fecha_generacion": "2025-10-05T02:00:00.000Z",
    "periodo": {
      "inicio": "2025-09-28T00:00:00.000Z",
      "fin": "2025-10-05T23:59:59.999Z"
    },
    "contenido_educativo": [...],
    "concursos_activos": [...],
    "ganadores_recientes": [...]
  }
}
```

### Enviar Digest

#### POST `/api/newsletter/send-digest`
Envía el digest a todos los suscriptores activos.

**Acceso:** CONTENT_ADMIN

**Body:** Objeto digest generado previamente

### Contenido Popular

#### GET `/api/newsletter/popular-content`
Obtiene contenido popular para incluir en el digest.

**Acceso:** CONTENT_ADMIN

**Query Parameters:**
- `limit`: Número de contenidos (default: 5)

## Ejemplos de Uso

### Flujo Completo de Suscripción

```javascript
// 1. Suscribirse al newsletter
const response = await fetch('/api/newsletter/subscribe', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: 'usuario@example.com' })
});

// 2. Confirmar suscripción (desde email)
await fetch('/api/newsletter/confirm', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ token: 'token-from-email' })
});
```

### Crear y Consumir Contenido Educativo

```javascript
// 1. Crear contenido (CONTENT_ADMIN)
const content = await fetch('/api/educational-content', {
  method: 'POST',
  headers: { 
    'Content-Type': 'application/json',
    'Authorization': 'Bearer token'
  },
  body: JSON.stringify({
    tipo: 'tutorial',
    categoria_multimedia: 'fotografia',
    titulo: 'Técnicas de Iluminación Natural',
    contenido: '...',
    nivel: 'intermedio',
    tiempo_lectura: 10,
    tags: ['iluminacion', 'natural'],
    estado: 'publicado'
  })
});

// 2. Obtener contenido público
const educationalContent = await fetch('/api/educational-content?categoria_multimedia=fotografia&nivel=intermedio');

// 3. Registrar vista
await fetch('/api/educational-content/track-view', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    contenido_id: 1,
    tiempo_lectura: 600,
    porcentaje_leido: 90
  })
});

// 4. Obtener recomendaciones (usuario autenticado)
const recommendations = await fetch('/api/educational-content/recommendations', {
  headers: { 'Authorization': 'Bearer token' }
});
```

### Generar y Enviar Digest

```javascript
// 1. Generar digest semanal
const digest = await fetch('/api/newsletter/generate-digest', {
  method: 'POST',
  headers: { 
    'Content-Type': 'application/json',
    'Authorization': 'Bearer admin-token'
  },
  body: JSON.stringify({
    fecha_inicio: '2025-09-28T00:00:00.000Z',
    fecha_fin: '2025-10-05T23:59:59.999Z',
    incluir_concursos: true,
    incluir_contenido_educativo: true,
    max_contenido: 5
  })
});

// 2. Enviar digest a suscriptores
await fetch('/api/newsletter/send-digest', {
  method: 'POST',
  headers: { 
    'Content-Type': 'application/json',
    'Authorization': 'Bearer admin-token'
  },
  body: JSON.stringify(digest.data)
});
```

## Códigos de Error

### Errores Comunes

- **400 Bad Request**: Datos de entrada inválidos
- **401 Unauthorized**: Usuario no autenticado
- **403 Forbidden**: Sin permisos suficientes
- **404 Not Found**: Recurso no encontrado
- **409 Conflict**: Conflicto (ej: email ya suscrito)
- **500 Internal Server Error**: Error interno del servidor

### Mensajes de Error Específicos

#### Newsletter
- `"Este email ya está suscrito al newsletter"` (409)
- `"Token de confirmación inválido"` (400)
- `"Esta suscripción ya está confirmada"` (400)
- `"Suscripción no encontrada"` (404)

#### Contenido Educativo
- `"Contenido educativo no encontrado"` (404)
- `"Este contenido no es de tipo educativo"` (404)
- `"No tienes permisos para realizar esta acción"` (403)

### Formato de Respuesta de Error

```json
{
  "success": false,
  "message": "Descripción del error",
  "errors": [
    {
      "field": "email",
      "message": "Email inválido"
    }
  ]
}
```

## Notas de Implementación

1. **Autenticación**: Las rutas protegidas requieren token JWT válido
2. **Permisos**: Solo usuarios con rol `CONTENT_ADMIN` o `ADMIN` pueden gestionar contenido
3. **Validación**: Todos los endpoints usan validación Zod para entrada de datos
4. **Paginación**: Las listas incluyen paginación con `page`, `limit`, `total` y `totalPages`
5. **Filtros**: Los filtros son opcionales y se pueden combinar
6. **Métricas**: El tracking de vistas es opcional pero recomendado para mejores recomendaciones
7. **Digest**: El envío de digest es simulado - requiere integración con servicio de email real
