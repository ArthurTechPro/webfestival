# Guía de Deployment - Sistema de Suscripciones

## 🚀 Preparación para Producción

### Requisitos Previos

#### Infraestructura
- **Node.js**: v18+ (recomendado v20+)
- **PostgreSQL**: v14+ con extensiones JSON
- **Redis**: v6+ para cache y sesiones
- **SSL/TLS**: Certificado válido para webhooks

#### Servicios Externos
- **Stripe**: Cuenta verificada con webhooks configurados
- **PayPal**: Cuenta business con API credentials
- **Email**: Servicio SMTP para notificaciones

## 🐳 Docker Deployment

### Dockerfile Optimizado

```dockerfile
# Multi-stage build para optimizar tamaño
FROM node:20-alpine AS builder

WORKDIR /app
COPY package*.json ./
COPY prisma ./prisma/

# Instalar dependencias y generar Prisma client
RUN npm ci --only=production && \
    npx prisma generate

# Etapa de producción
FROM node:20-alpine AS production

# Instalar dumb-init para manejo de señales
RUN apk add --no-cache dumb-init

# Crear usuario no-root
RUN addgroup -g 1001 -S nodejs && \
    adduser -S webfestival -u 1001

WORKDIR /app

# Copiar archivos necesarios
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/prisma ./prisma
COPY . .

# Compilar TypeScript
RUN npm run build

# Cambiar ownership
RUN chown -R webfestival:nodejs /app
USER webfestival

# Exponer puerto
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:3000/api/v1/health || exit 1

# Usar dumb-init como PID 1
ENTRYPOINT ["dumb-init", "--"]
CMD ["npm", "start"]
```

### Docker Compose para Producción

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://postgres:${POSTGRES_PASSWORD}@db:5432/webfestival
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db
      - redis
    restart: unless-stopped
    deploy:
      replicas: 3
      resources:
        limits:
          memory: 512M
          cpus: '0.5'
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/api/v1/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_DB=webfestival
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data

      resources:

          memory: 1G

      - "80:80"

      - ./nginx.conf:/etc/nginx/nginx.conf

      - app
```

```yaml
          httpGet:
            path: /api/v1/health

  name: webfestival-api-service

    targetPort: 3000
```

## 🔧 Configuración de Webhooks

### Stripe Webhooks

1. **Configurar endpoint en Stripe Dashboard:**
   ```
   URL: https://your-domain.com/api/v1/subscriptions/webhooks/stripe
   Eventos: invoice.payment_succeeded, customer.subscription.updated, etc.
   ```

2. **Verificar configuración:**
   ```bash
   curl -X POST https://your-domain.com/api/v1/subscriptions/webhooks/stripe \
     -H "Stripe-Signature: test_signature" \
     -d '{"type": "test"}'
   ```


## 🔒 Seguridad en Producción

### SSL/TLS Configuration

```nginx
server {
    listen 443 ssl http2;
    server_name api.webfestival.com;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;

    location /api/v1/subscriptions/webhooks/ {
        proxy_pass http://app:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 🗄️ Backup y Recuperación

### Backup Automático de Base de Datos

```bash
#!/bin/bash
# backup-db.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backups"
DB_NAME="webfestival"

# Crear backup
pg_dump $DATABASE_URL > $BACKUP_DIR/webfestival_$DATE.sql

# Comprimir
gzip $BACKUP_DIR/webfestival_$DATE.sql

# Limpiar backups antiguos (mantener 7 días)
find $BACKUP_DIR -name "webfestival_*.sql.gz" -mtime +7 -delete

echo "Backup completado: webfestival_$DATE.sql.gz"
```

### Cron Job para Backups

```bash
# Agregar a crontab
0 2 * * * /path/to/backup-db.sh >> /var/log/backup.log 2>&1
```

## 🚀 Proceso de Deployment

### CI/CD Pipeline (GitHub Actions)

```yaml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '20'
      - run: npm ci
      - run: npm test

  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Build Docker image
        run: docker build -t webfestival/api:${{ github.sha }} .
      
      - name: Push to registry
        run: |
          echo ${{ secrets.DOCKER_PASSWORD }} | docker login -u ${{ secrets.DOCKER_USERNAME }} --password-stdin
          docker push webfestival/api:${{ github.sha }}
      
      - name: Deploy to Kubernetes
        run: |
          kubectl set image deployment/webfestival-api api=webfestival/api:${{ github.sha }}
          kubectl rollout status deployment/webfestival-api
```

### Deployment Steps

1. **Pre-deployment:**
   ```bash
   # Ejecutar migraciones
   npm run migrate:deploy
   
   # Verificar salud de la aplicación
   npm run health-check
   ```

2. **Deployment:**
   ```bash
   # Blue-green deployment
   kubectl apply -f k8s/
   kubectl rollout status deployment/webfestival-api
   ```

3. **Post-deployment:**
   ```bash
   # Verificar métricas
   curl https://api.webfestival.com/metrics
   
   # Smoke tests
   npm run test:smoke
   ```

## 🔍 Troubleshooting

### Logs Comunes

```bash
# Ver logs de la aplicación
kubectl logs -f deployment/webfestival-api

# Logs de base de datos
kubectl logs -f statefulset/postgres

# Métricas de sistema
kubectl top pods
```

### Problemas Frecuentes

1. **Webhook timeouts:**
   - Verificar conectividad de red
   - Aumentar timeout en Stripe/PayPal
   - Revisar logs de aplicación

2. **Database connection issues:**
   - Verificar pool de conexiones
   - Revisar configuración de red
   - Comprobar límites de PostgreSQL

3. **Memory leaks:**
   - Monitorear métricas de memoria
   - Revisar garbage collection
   - Analizar heap dumps
