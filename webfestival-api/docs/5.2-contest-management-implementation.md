## Funcionalidades Implementadas

### 1. CRUD Completo para Concursos (Administradores)

**Endpoints de Administración:**
- `POST /api/v1/concursos` - Crear nuevo concurso
- `GET /api/v1/concursos` - Obtener todos los concursos con filtros
- `GET /api/v1/concursos/:id` - Obtener concurso específico
- `PUT /api/v1/concursos/:id` - Actualizar concurso
- `DELETE /api/v1/concursos/:id` - Eliminar concurso

**Características:**
- Validación completa de datos con Zod
- Control de fechas (fecha final debe ser posterior a fecha inicial)
- Gestión de estados: PROXIMAMENTE, ACTIVO, CALIFICACION, FINALIZADO
- Configuración de límites (máximo envíos, tamaño máximo de archivos)
- Protección contra eliminación de concursos con inscripciones o medios

### 2. Sistema de Inscripciones para Participantes

**Endpoints de Inscripción:**
- `POST /api/v1/concursos/inscripcion` - Inscribirse a un concurso
- `DELETE /api/v1/concursos/inscripcion/:concursoId` - Cancelar inscripción
- `GET /api/v1/concursos/mis-inscripciones` - Obtener mis inscripciones
- `GET /api/v1/concursos/:concursoId/verificar-inscripcion` - Verificar estado de inscripción

**Validaciones de Inscripción:**
- Concurso debe estar activo
- Período de inscripción no debe haber finalizado
- No permitir inscripciones duplicadas
- No permitir cancelar inscripción si ya se enviaron medios
- No permitir cancelar en concursos finalizados o en calificación

### 3. Endpoints Públicos para Consulta

**Endpoints Públicos:**
- `GET /api/v1/concursos/activos` - Obtener concursos activos
- `GET /api/v1/concursos/finalizados` - Obtener concursos finalizados (paginado)
- `GET /api/v1/concursos/:id` - Obtener detalles de concurso específico

**Características:**
- Acceso sin autenticación para visualización pública
- Filtrado automático por estado y fechas
- Información completa de categorías, inscripciones y medios
- Paginación en concursos finalizados

## Manejo de Errores

### Errores de Validación
- **400 Bad Request**: Datos inválidos, fechas incorrectas
- **401 Unauthorized**: Token de autenticación requerido
- **403 Forbidden**: Permisos insuficientes (no es ADMIN)
- **404 Not Found**: Concurso no encontrado
- **409 Conflict**: Inscripción duplicada, concurso con datos existentes
